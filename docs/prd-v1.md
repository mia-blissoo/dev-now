# DEV TEAM LAUNCHPAD — PRD

> BLISSOO DEV TEAM 내부 프로젝트 현황판
> 작성일: 2026-06-01 · 상태: Draft v1

## 1. 개요
BLISSOO 개발팀(JISOO APP 담당)이 만든/만들고 있는 것들을 한 화면에서 보여주는 내부용 모바일 웹. C레벨·타팀이 개발팀의 현재 진행 상황과 실험(POC) 결과물을 별도 보고 없이 바로 확인할 수 있게 하는 게 목적.

## 2. 배경 / 문제
- BLISSOO는 팀마다 소통툴이 다름 (개발팀 Slack, 그 외 카톡 등) → 전체 현황을 한눈에 보는 통로가 없음.
- 개발팀의 산출물·실험이 흩어져 있어 외부(C레벨·타팀)에 잘 보이지 않음.
- 보고 단계(현 워크플로우 6단계)가 수동이라 매번 사람이 정리해야 함.

## 3. 목표 / 비목표
**목표**
- 개발팀 진행 상황과 POC를 외부에 "별도 보고 없이" 노출.
- 모바일에서 빠르게 열어보는 경량 현황판.
- 데이터(태스크/POC)만 갈아끼우면 화면이 갱신되는 구조.

**비목표 (현 단계)**
- 인증/권한 관리 (내부 링크 공유로 충분).
- 실시간 자동 동기화 (1차는 수동 갱신, 추후 자동화).
- 프로젝트 관리 도구 대체 (Jira 대체 X, 요약 뷰 역할).

## 4. 사용자
- 1차: C레벨/부회장 — 현황 훑어봄.
- 2차: 타팀 — 개발팀이 뭘 하는지 파악.
- 운영자: 개발팀(PM) — 데이터 갱신 담당.

## 5. 정보 구조 (GNB 3탭)
| 탭 | 내용 | 형태 |
|---|---|---|
| DEV NOW | 현재 JISOO APP 진행 상황 | 간트차트 (좌우 스크롤, TASK 컬럼 고정, TODAY 라인) |
| POC | 개발팀이 실험한 결과물 (슈몬타로, 슈몬포토 등) | 카드뷰 (상태: POC) |
| Daily Enter | K-엔터 뉴스·이슈 모니터링 | 외부 링크 (k-ent-pulse.vercel.app, 새 탭) |

### 5.1 DEV NOW (간트)
- 태스크별 시작/종료일·상태(완료/진행중/예정/마일스톤)를 가로 막대로 표현.
- 워크플로우 단계(기획 → 디자인 → iOS/AOS 개발 → QA → 심사 → 릴리즈)를 그대로 행으로.
- 오늘 날짜에 세로 기준선(TODAY).

### 5.2 POC (카드)
- 카드 1개 = POC 1건. 제목 · 한 줄 설명 · 태그 · 링크(선택).
- 빈 슬롯("+ POC 추가")으로 확장 유도.

### 5.3 Daily Enter
- 콘텐츠 패널 없이 탭 클릭 시 외부 URL로 바로 이동.

## 6. 데이터 모델
화면은 코드 상단의 두 데이터 객체로만 구동됨 (디자인/로직 수정 없이 내용만 갱신).

```js
GANTT = {
  rangeStart, rangeEnd,
  tasks: [{ name, start, end, status }]   // status: done | progress | todo | milestone
}

POCS = [{ name, desc, tag, link }]
```

## 7. 디자인
- 방향: 모노톤(블랙 베이스 #0d0d0d), 컬러 강조 없음.
- 타이포: DM Mono (모노스페이스 통일).
- 레이아웃: 모바일 기준 반응형, 최대 폭 540px 중앙 정렬.
- 모션: 카드 스태거 등장, 라이브/진행 상태 표시, 호버 인터랙션.

## 8. 기술
- 정적 단일 HTML 파일 (HTML/CSS/Vanilla JS, 외부 의존성 없음 — 폰트만 Google Fonts).
- 배포: 정적 호스팅(예: Vercel) 또는 사내 링크.
- 관련 프로젝트 Daily Enter는 Next.js (/Users/miachoi/claude-code/k-ent-pulse).

## 9. 향후 (자동화 로드맵)
1. DEV NOW ← Jira 이슈/스프린트에서 태스크·상태 자동 동기화.
2. POC ← 레포/배포 목록에서 카드 자동 생성.
3. 릴리즈/보고 ← 릴리즈 노트 자동 작성 → C레벨 보고 연동.
→ 궁극적으로 LAUNCHPAD가 워크플로우 6단계의 "보고" 단계를 대체.
