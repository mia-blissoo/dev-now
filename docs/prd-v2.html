2-2. 편집자 데이터 갱신 (간트 vs 텍스트)

A1
간트차트 (수동)
편집자가 DEV NOW 페이지 EDIT 모드 진입 → 드래그/resize/상태 변경 → 즉시 저장.
B1
텍스트 콘텐츠 (자동)

B2
갱신 트리거
슬랙 List 수정 시 webhook 또는 폴링으로 동기화. (미정: 주기 vs 실시간 — 5번 항목 참조)

3. 화면·기능 명세

3-1. 비로그인 랜딩 (Confidential View)

  • 상단 헤더에 "Confidential — BLISSOO 내부 전용" 서브헤더 노출
  • DEV NOW 탭의 간트차트만 흐릿하게(opacity 0.5) 미리보기로 표시
  • POC / Daily Enter 탭 접근 시 로그인 모달
  • 모든 카드·간트 막대 클릭 시 → "Google로 로그인" 모달
  • 우측 상단 "Google로 로그인" CTA 버튼 상시 노출

3-2. POC 탭 서브헤더

Proof of Concept
개발팀이 본 제품에 들어가기 전 빠르게 검증한 실험 결과물. 정식 출시되지 않은 프로토타입·MVP 등이 모여있어요.

3-3. 카드 모달 상세

  • 카드 클릭 시 화면 중앙에 모달 (V1의 하단 바텀시트 → V2 모달로 변경)
  • 모달 사이즈: 데스크탑 720px / 모바일 풀스크린
  • 구성: 제목, 상태 칩, 담당자, 메모, 데드라인, 첨부 이미지(여러 장 캐러셀), 외부 링크 버튼
  • 외부 링크가 있으면 "원문 보기" 또는 "바로가기" 버튼 — 새 탭으로 열림
  • 편집자만 우상단에 "편집" 버튼 노출 → 클릭 시 편집 폼으로 전환
  • 뷰어는 읽기 전용

3-4. "부회장님 확인 필요" 별표

  • 카드별로 연보라색 별표 아이콘(★) 토글 가능 — 카드 우상단 또는 제목 옆에 표시
  • 의미: "부회장님 의사결정 필요 / 컨펌 받아야 하는 항목"
  • 색: #C4B5FD 또는 #A78BFA 계열 (JISOO APP 보라와 어울리는 톤)
  • 토글 권한: 편집자 + 관리자만 추가/제거 가능. 뷰어는 별표 시각만 보임
  • 위치: 메인 간트 행 좌측 라벨 옆, 모달 상세 제목 옆 둘 다 표시
  • 필터: 우상단에 "별표만 보기" 토글 (모든 권한 가능)
  • 슬랙 List에는 없는 DEV NOW 자체 메타 → 슬랙 sync 시 보존

3-5. 어드민 — 가입 승인 화면

  • 관리자만 접근 가능. URL: /admin 또는 우상단 메뉴
  • 가입 요청 목록 (이메일, 이름, 요청 시각, 도메인)
  • 각 요청에 [뷰어 승인] [편집자 승인] [거절] 버튼
  • 승인됨 / 거절됨 / 대기중 필터
  • 승인 시 신청자 이메일로 자동 안내 (옵션)

5. 데이터 소스

간트차트 (수동 편집)

  • 편집자가 EDIT 모드에서 직접 조작
  • 드래그앤드롭으로 순서 변경
  • 막대 양 끝 resize로 일정 조절
  • 막대 클릭 → 상태(done/progress/todo) 변경
  • 저장: localStorage(V1) → DB(V2 미정, 9-1 항목 참조)

텍스트 콘텐츠 (슬랙 자동 연동) 김종원 연동 완료

  • 소스: 슬랙 List F0ASC4WB3V4 (JISOO APP Plan)
  • 대상: 태스크명, note, deadline, 상태 필드
  • 흐름: 슬랙 List 수정 → dev-now bot 감지 → DEV NOW 갱신
  • 방향: 슬랙 → DEV NOW 단방향 (V2)
  • 양방향 동기화는 V3 검토

6. 보안·접근 정책

  • HTTPS 전용 (Vercel 기본 제공)
  • Google OAuth 클라이언트 ID 발급 + 허용 도메인 화이트리스트 (옵션)
  • JWT 또는 세션 쿠키로 인증 유지 (httpOnly · secure)
  • 관리자 작업(승인·권한 변경)은 서버에서 권한 재검증
  • 비로그인 사용자에게 노출되는 데이터: 카드명·간트 막대 윤곽만. 메모·담당자·링크 등은 모달 안에 있으므로 미노출
  • 로그아웃 시 토큰 즉시 폐기 + localStorage 캐시 비움

8. 결정 매트릭스

1. 백엔드·DB 인프라
결정: 개발팀 결정 위임 — PM은 관여 안 함. 개발팀이 Supabase / Vercel KV / Firebase 등 자율 선택
2. 슬랙 List 동기화 주기
결정: 매일 (daily) — 하루 1회 batch sync. 새벽 또는 출근 전 정기 실행. 실시간 webhook은 V3
3. 모달에서 편집 가능 여부
결정: 옵션 B (모달 안 인라인 편집) — 편집자 권한일 때 모달 우상단 "편집" 버튼 → 같은 모달에서 폼 전환. 슬랙 sync 필드(name, pjt, dev, note)는 read-only로 잠금
4. 비로그인 사용자가 보는 정보의 깊이
결정: 옵션 B + 강한 블러 — 카드명·간트 막대 윤곽 노출하되 blur(8px) 이상 강하게 처리. 무엇이 있다는 신호만 주고 텍스트는 절대 안 읽히게
5. 가입 승인 후 알림
결정: 옵션 A (알림 없음) — 가입자가 직접 다시 접속해서 확인. V2는 최소 기능
6. 슬랙 List ↔ DEV NOW 데이터 정책
결정: 슬랙이 source of truth — DEV NOW의 기존 데이터는 매 sync마다 슬랙 기준으로 덮어쓰기 OK. 단 DEV NOW 자체 메타(별표·간트 일정·phases·lifecycle 등 슬랙에 없는 필드)는 보존
7. 디자인 톤
결정: 옵션 A (다크 톤 유지) — V1 디자인 그대로. 라이트 모드 X

9. 법적·운영 체크리스트

  • 내부 사용 도구지만 외부 도메인(vercel.app)에 배포되므로 confidential 명시 필요
  • 가입 신청 시 약관·개인정보 동의 (이메일·이름만 수집) — 1줄 안내로 충분
  • 로그·접근 기록 보관 (누가 언제 무엇을 편집했는지) — V3
  • 관리자 권한 박탈 / 계정 비활성화 절차 — 어드민 화면에서
  • 슬랙 List 데이터를 외부에 노출하지 않음 (인증 통과한 사용자에게만)

용어 정리

POC (Proof of Concept)
개발팀이 본 제품에 들어가기 전 빠르게 검증한 실험 결과물. 정식 출시되지 않은 프로토타입·MVP 등.
슬랙 List
슬랙의 데이터베이스형 보드 기능. 행마다 필드(컬럼)를 가짐. 본 PRD에서는 "JISOO APP Plan" 보드(F0ASC4WB3V4)를 지칭.
dev-now bot
슬랙 List 변경을 감지하여 DEV NOW에 반영하는 자동화 봇. 김종원이 셋업 완료. #jisoo-v2 채널에 추가됨.
Google OAuth
Google 계정으로 로그인하는 표준 인증 방식. 사용자가 비밀번호를 별도로 만들지 않아도 됨.
역할 (role) / 상태 (status)
role은 권한 레벨(viewer/editor/admin), status는 가입 상태(pending/approved/rejected). 둘은 독립적 — 승인된 뷰어, 거절된 편집자 등 조합 가능.
